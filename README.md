📉 Customer Churn Analysis

End-to-end data analytics project focused on predicting and understanding customer churn using Python, SQL, and Power BI.

📌 Project Overview

This project analyzes telecom customer data to identify key factors leading to customer churn. By examining customer demographics, service usage, contract types, and billing information, the project uncovers patterns that help businesses improve customer retention and lifetime value.

The dataset contains 7,000+ customer records with detailed information about subscriptions, services, and payment behavior.

🎯 Business Objectives
Identify the main drivers behind customer churn
Analyze differences between churned and retained customers
Understand the impact of contract type, tenure, and services on churn
Detect high-risk customers likely to churn
Provide actionable recommendations to reduce churn

🔍 Analysis Workflow
1️⃣ Data Cleaning & Preparation
Converted Churn column from object → numerical (Yes = 1, No = 0)
Fixed TotalCharges datatype (object → float)
Handled missing and blank values
Standardized categorical variables
Prepared dataset for analysis in Python & SQL
2️⃣ Exploratory Data Analysis (EDA)
Churn distribution analysis
Tenure vs churn relationship
Monthly charges vs churn
Contract type comparison
Service usage impact on churn
3️⃣ Churn Behavior Analysis
Identified high churn in low tenure customers
Compared churn across contract types
Analyzed impact of value-added services
Studied relationship between payment methods and churn
📈 Key Insights
Customers with month-to-month contracts have the highest churn rate
Low tenure customers are more likely to churn early
Customers without online security, backup, or tech support churn more
Higher monthly charges increase churn risk
Long-term contracts significantly improve retention
💡 Business Recommendations
Encourage customers to shift to long-term contracts
Provide bundled services (security, backup, support)
Target new customers with early retention strategies
Optimize pricing for high monthly charge users
Offer incentives for high-risk churn segments
🧠 Key Learnings
Importance of data cleaning (especially datatype issues like TotalCharges)
Real-world churn patterns in subscription-based businesses
Combining multiple features to identify churn drivers
Translating analytical insights into business strategies
Hands-on experience with telecom dataset
